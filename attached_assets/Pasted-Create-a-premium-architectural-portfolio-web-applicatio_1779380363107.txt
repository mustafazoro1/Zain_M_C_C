Create a premium architectural portfolio web application using React (Vite) on the frontend, FastAPI on the backend, and a PostgreSQL database. The site design must feature an ultra-minimalist luxury dark/monochrome aesthetic using Tailwind CSS (obsidian backgrounds, crisp slate borders, explicit editorial typography).

1. NO-REVENUE EDITING ADMIN PANEL (Core Focus)
Build a dedicated, password-protected dashboard route (/admin) designed EXCLUSIVELY as a content editor and media management system for a high-end construction/portfolio firm. Completely omit all revenue graphs, user analytics, and generic dashboard widgets. It must focus entirely on clean, functional CRUD operations:
- Project Creation Wizard: A step-by-step form to add a brand new construction project. It must handle raw text data alongside a media gallery uploader.
- Media Matrix Manager: Inside each project editor, create a dynamic media field. The admin must be able to add multiple image URLs, drag or re-order them, and use a simple toggle switch or radio button to explicitly mark exactly ONE image as the "is_hero" image (the thumbnail used for grids).
- Live Content Table: A simple list view showing all current live projects with quick buttons to 'Edit Details', 'Manage Images', or 'Unpublish/Delete'.

2. RELATIONAL DATABASE SCHEMA (PostgreSQL via SQLAlchemy)
Build the database tables with strict columns matching these architectural specifications:
- 'categories': id, name, slug
- 'projects': id, category_id, title, slug, location, client, sector, size, scope, status, long_description
- 'project_images': id, project_id, image_url, is_hero (boolean)

3. FRONTEND VIEW & REVEAL INTERACTION
- Global Custom Cursor: A clean, circular cursor following the mouse using Framer Motion. When hovering over clickable project layouts, the circle expands smoothly.
- Homepage Accordion: Full-screen equal-width vertical panels. On hover, the selected panel expands dramatically via flexbox, and the project's title and primary image reveal smoothly from the center outward.
- Project Detail Layout: Clicking a project opens a sleek case study view rendering exactly in this order:
  * Section A: Full-width Hero Image + Title.
  * Section B: Two-Column Split Layout. Left column displays a clean typographic list of metadata: Location, Client, Sector, Size, Scope, Status. Right column displays the 'long_description' text narrative.
  * Section C: High-resolution vertical/masonry scrolling grid of all secondary project images.

4. WHERE THE AGENT SHOULD USE ITS OWN BRAIN (Reference: rmjm.com)
- UI Flow & Layout Nuances: Use your own judgment to adapt rmjm.com's clean architectural grid scaling. Implement smooth, staggered transitions so that when text elements open from the center during a hover state, they fade and slide upward with premium easing curves instead of just snapping into place.
- Admin UI Elegance: Design the admin editor using the same luxury dark aesthetic as the frontend. The input forms, image slot cards, and save buttons should look like a bespoke internal utility built for an elite design studio, rather than a generic white bootstrap template. Use clean borders and sleek hover feedback for image deletion/reordering slots.
